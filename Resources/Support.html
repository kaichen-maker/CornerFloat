Support

CornerFloat for macOS

Fast recovery: use the menu-bar CornerFloat icon even when every panel is hidden. Press ⇧⌘Space by default to show or hide all panels; choose another preset under Settings → Global Shortcut if it conflicts. Choose Quit CornerFloat to stop the app completely.

A website will not load or sign in

  • Check the in-panel error message. Choose Try Again only when it is offered; CornerFloat never silently replays a failed form POST.
  • Open the page in your default browser to confirm the service is online.
  • OAuth popups open in a new CornerFloat tab when the provider permits embedded sign-in. Passkeys use a separate macOS system authorization flow.
  • CornerFloat permits ChatGPT's Google redirect to continue in the panel and identifies itself truthfully as CornerFloat. Google may still reject OAuth in WKWebView; if it does, use More → Open in Default Browser. That browser's Cookie session remains separate.

A Quick Site does not open

Open Windows & Library → Quick Sites and confirm that the alias is unique and the destination is a complete HTTP or HTTPS URL. Shortcut words ignore capitalization, spaces, accents, and character width. If no alias matches, CornerFloat treats the text as a normal search.

A website cannot use a passkey

A local source build hides the release-only Passkey action. In an eligible signed release, CornerFloat never requests passkey access at launch. Choose CornerFloat → Enable or Review Passkey Access…. If access is off, open System Settings → Privacy & Security → Passkeys Access for Web Browsers, enable CornerFloat, and reload the sign-in page. If the setting is unavailable on a managed Mac, contact its administrator.

A public build also needs the Apple-approved Web Browser Public Key Credential entitlement, Developer ID signing, and a matching embedded Developer ID provisioning profile. Automated source tests verify the authorization-state logic without exposing an unusable menu, but an ad-hoc build is not a valid end-to-end cross-site passkey test.

Website voice or dictation does not start

  1. Confirm the address begins with https://. CornerFloat denies microphone requests from insecure pages and denies every camera or combined camera-and-microphone request.
  2. Click the website’s voice, dictate, or microphone control again. CornerFloat itself does not request microphone access at launch, although a website can choose when its code asks for access.
  3. Respond to both the macOS CornerFloat microphone prompt and the requesting website’s own WebKit microphone prompt in the order they appear. These are separate decisions and neither is approved automatically.
  4. If macOS access was previously denied, quit CornerFloat, open System Settings → Privacy & Security → Microphone, enable CornerFloat, reopen it, and reload the HTTPS page.
  5. If access is already enabled, review the website’s own microphone or site settings, account requirements, and service status. Test the same feature in the default browser to distinguish a site problem from an app problem.

CornerFloat does not itself record, store, or upload audio. Once both permissions are granted, the selected website handles microphone audio under its own privacy policy. Stop voice mode on the website when finished. If a feature requires a camera, use a browser that supports that request because CornerFloat keeps camera capture outside its current scope.

Bluetooth voice sounds deep, slow, or delayed

Apple explains that using a Bluetooth headset for both playback and microphone input switches it from high-quality listening to lower-quality two-way audio. Before WebKit presents an HTTPS website’s microphone decision on a risky Bluetooth input/output route, CornerFloat offers three choices when a built-in input is available:

  • Use Mac Microphone temporarily changes the system default input to the built-in microphone, then continues to the website’s WebKit decision. This happens only after you click the button.
  • Continue with Bluetooth leaves the current input and output unchanged and continues to the website’s WebKit decision.
  • Cancel denies the current website microphone request.

If no built-in microphone is available, the explanation offers only Continue with Bluetooth and Cancel.

After a temporary switch, CornerFloat makes a best-effort attempt to restore the previous input when the website stops microphone capture, the panel closes, or the app quits. It restores only if the default input is still the device CornerFloat selected, and it relinquishes restoration ownership when it observes a later default-input change. The temporary default-input change is system-wide and can also be seen by other audio apps.

If audio remains distorted, end the website voice session, choose the Mac’s built-in microphone under System Settings → Sound → Input, and start again. Include the selected input and output device names in a support report, but never include a recording of a private conversation.

A panel is hard to find

Use Window → Move to Bottom Right, or reopen a saved workspace. CornerFloat repairs off-screen panel positions after display changes and wake. If edge auto-hide is enabled, move the pointer to the screen edge to reveal the panel.

Back up or move the library

Use Settings → Local Data to export, import, or reveal the versioned JSON library. Import validates and previews the file, then asks before replacing favorites, recents, Quick Sites, and saved workspaces. It does not close panels or alter website cookies and sign-ins.

Reset local data

Quit CornerFloat and remove ~/Library/Application Support/CornerFloat to delete Quick Sites, favorites, recents, and saved workspaces. To reset app preferences too, run defaults delete com.calvinkai.cornerfloat while CornerFloat is quit. WebKit cookies and website data are separate; remove the app’s WebKit data from your macOS user Library only if you also want to clear website sessions. These actions cannot be undone.

Information to include in a support request

Include the CornerFloat version and build number, macOS version, Mac model/architecture, the page or feature involved, exact steps, and a screenshot with private information removed. The app’s Support window provides a Copy Diagnostics button that gathers only these non-secret system details.

Security

Do not include passwords, passkeys, session cookies, authentication codes, or private screenshots in a report. Verify that public releases are Developer ID signed and notarized before installing.
